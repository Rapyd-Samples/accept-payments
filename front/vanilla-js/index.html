Create Checkout Page

            

                
                

                    
Inputs

                    

                        
                        

                            The amount of the payment, in units of the currency defined in
                            currency. Required if cart_items is not used. Decimal.
                        


                        
                        

                            The two-letter ISO 3166-1 ALPHA-2 code for the country.
                        


                        
                        

                            Defines the currency for the amount. Three-letter ISO 4217 code.
                            For FX transactions, see Checkout Page Object.
                        


                        
                        

                            ID of the 'customer' object. String starting with cus_. When
                            used, the customer has the option to save card details for
                            future purchases. This field is required for certain production
                            mode payment methods.
                        




                        
                        

                            URL where the customer is redirected when payment is successful.
                            Relevant to bank redirect payment methods. Localhost addresses are not
                              valid
                        


                        
                        

                            URL where the customer is redirected after pressing Finish to
                            exit the hosted page. Localhost addresses are not valid
                        


                        
                        

                            URL where the customer is redirected when payment is not
                            successful. Relevant to bank redirect payment methods. Localhost addresses are
                              not valid
